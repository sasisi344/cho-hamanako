# ポイント記事 SEO / SXO 監査（エージェント用）

`category: points` の釣り場ピラーページを、検索・構造化データ・読者体験の観点で点検し、**修正タスクに落とし込みやすくする**ための手順書です。

## 優先参照（必読順）

1. [.agents/point-article/SKILL.md](./SKILL.md) — 標準構成・ライティングルールの正本
2. [.agents/structured-data-rules.md](../structured-data-rules.md) — Frontmatter・URL・breadcrumb
3. [CLAUDE.md](../../CLAUDE.md) — 内部リンク・`BlogCard` / `TackleCard` の実在確認ルール
4. 必要に応じて [.agents/writing-style/SKILL.md](../writing-style/SKILL.md)

## 適用対象

- パス例: `src/content/blog/points/**/index.mdx`（およびサブエリア配下のポイント記事）
- **新規執筆**と**既存記事のリライト**の両方で使用する

---

## エージェント実行フロー（推奨）

1. **対象ファイルを開き**、Frontmatter → 見出し（H2/H3）→ 本文の順に読む。
2. 下記 **チェックリスト** で該当項目に `OK` / `要修正` / `該当なし` を付ける（推測で埋めない）。
3. **要修正** だけを、末尾の **出力テンプレート** に転記する。
4. 修正実装時は **P0 → P1 → P2** の順。`BlogCard` / 施設数値は必ずデータソース確認（CLAUDE.md）。

---

## P0 — インデックス・データ整合（検索・構造化の土台）

| ID | 確認内容 | 合格基準 |
| :--- | :--- | :--- |
| P0-1 | `noindex` | `false`（意図的な非公開でないこと） |
| P0-2 | `slug` / ファイル配置 | [structured-data-rules.md](../structured-data-rules.md) の URL 方針と矛盾しないこと |
| P0-3 | `location.lat` / `lng` / `googleMapUrl` | 数値・URL が本文の `Map` / `GMapButton` と一致 |
| P0-4 | `fishinginfo` と本文 | `bestSeason` / `bestMonths` / `targetFish` / `methods` に書いた季節・魚種・釣法が、本文で**最低1段落以上**触れられている（未記載の季節・魚種を並べない） |
| P0-5 | `facilities` と本文 | 駐車・トイレ・夜釣り等の主張が本文と矛盾しない |
| P0-6 | `summary` | **検索結果・SNS用の要約**として機能している。文字数の自慢（「◯文字超」等）だけに頼らない。SKILL の目安 **70〜120字程度**（サイトのテンプレートが異なる場合は `config` / レイアウト側の仕様に合わせる） |
| P0-7 | `title` | 地名・ベネフィットが分かる。**釣果保証に見える過剰表現**（例: 断定の「爆釣」）は避け、必要なら疑問形や「狙い方」寄りにする |

## P1 — 標準構成（ピラーの情報設計）

[SKILL.md の「標準構成」](./SKILL.md) と突き合わせ、**該当 H2 が存在するか**を確認する。統合している場合は **どの見出しに統合したか** を出力テンプレに明記する。

| 標準 H2（要旨） | よくある不足 | 修正のヒント |
| :--- | :--- | :--- |
| ポイント概要と詳細アクセス | アクセスが粗い、釣具店が一行 | 駐車の入り方・台数・料金、コンビニ／釣具店を **リスト化**（可能なら営業時間・定休・特徴。データはマスター参照） |
| 水中地形と構造 | 説明が短い | 足元〜沖、岸の種類、潮・川の影響を **エリア分け（H3）** で整理 |
| シーズン別の狙い目とメインターゲット | 並列のみで主役がない | **イチオシ魚種を1つ深掘り** し、その他は簡潔に。関連 `BlogCard` を配置 |
| 推奨タックルと基本的な仕掛け | **セクション欠落**が多い | ロッド・ライン・代表ルアー／仕掛けを **このポイント向けに** 具体化（店販カードは使わない） |
| 安全と毒魚 | 表・中奥の区別がない | エリアに応じ [SKILL の毒魚ルール](./SKILL.md) に沿って記述 |
| 独自のローカルルールとマナー | 安全と混在 | 住民・歩行者・漁業・駐車ルールを **独立 H2** で整理できるなら分離 |
| あなたにぴったりの攻略ルート | 欠落 | 初心者向け／深掘りなど **読者分岐** と内部リンク |
| まとめ | マナー呼びかけが弱い | 必要に応じ `> [!IMPORTANT]` で締め |

---

## P2 — 本文品質・SXO・内部リンク

| ID | 確認内容 | 合格基準 |
| :--- | :--- | :--- |
| P2-1 | 見出しの番号 | SKILL どおり **見出しに「1. 2. 3.」形式の連番を振らない**（本文リストの番号は可） |
| P2-2 | 見出しの装飾 | 絵文字のみに依存せず、**テキストだけで主題が伝わる** H2 を心がける（絵文字は任意） |
| P2-3 | 煽り・最上級語 | 「最強」「NO.1」「圧倒的」「爆釣」の連発を避け、**条件付き・体験ベース**に置き換え可能か検討 |
| P2-4 | 太字 | `**` は使わず `<strong>...</strong>`（SKILL 厳守） |
| P2-5 | アフィリエイト | ピラーページ本文に `TackleCard` 等を置いていない |
| P2-6 | 内部リンク | `BlogCard` の `slug` は `src/content/blog/` に実在。推測で slug を書かない |
| P2-7 | 表記ゆれ | SKILL の地名表や、同一ポイント内での名称の一貫性 |
| P2-8 | MDX の無駄 | 未使用の `import` やコンポーネントがない |
| P2-9 | 導入 | 「釣！浜名湖へようこそ」から始まる（SKILL）。**文字数自慢**で終わらない |

## 修正時の出力テンプレート（コピー用）

以下をそのまま埋めて PR 説明・Issue・エージェントへの指示に使う。

```markdown
## 対象
- ファイル: `src/content/blog/points/.../index.mdx`
- slug:

## P0
- [ ] P0-4 フロントマターと本文の季節・魚種の整合:
- [ ] P0-5 施設情報の整合:
- [ ] P0-6 summary:
- [ ] P0-7 title:

## P1（欠けている標準 H2）
- [ ] 推奨タックルと基本的な仕掛け:
- [ ] あなたにぴったりの攻略ルートは？:
- [ ] その他:

## P2
- [ ] 煽り語・見出し番号・未使用 import 等:

## 次アクション（箇条書き）
1.
2.
```

## 備考：検索意図との対応（リライト時の観点）

- **情報意図**: アクセス・駐車・トイレ・夜釣り可否・ターゲット魚 → Frontmatter と H2 で明示。
- **比較・周辺意図**: 近隣ポイントや家族向け記事へ `BlogCard` で導線。
- **信頼**: 釣果の断定より、**地形・季節・リスク・マナー**を具体的に書くほど E-E-A-T 的に有利になりやすい。

このファイルは **ルールの複製ではなく監査用** とする。執筆の詳細ルールは常に [SKILL.md](./SKILL.md) を正とする。
