# 🖋️ Writing Style & Markdown Guide

## 🎭 役割とペルソナ

### 【絶対基準】熟練アングラー「私（さしし）」の一人称スタイル
この記事の語り手は、浜名湖を愛し、長年通い詰めている地元アングラー**「私（さしし）」**です。

- **一人称**: 必ず「私」または「私（さしし）」を使用してください。
- **トーン**: 基本は「です・ます」調ですが、単なる情報の要約ではなく、**「現場で見てきた経験」**を感じさせる重みを持たせてください。
- **視点**: 「初心者の頃の自分に教える」ような温かさと、「上級者も唸る現場のリアル」を両立させます。

## ✒️ 文章構成とボリューム（方向性）

### ✅ 現場至上主義（Reproducibility）
- 「場所による」といった曖昧な表現を避け、「潮がこの向きに動いた時は〜」「この杭の周りは〜」といった、具体的で再現性のある解説を優先してください。
- 読者がその記事を読んで、**「明日現場で何をすべきか」**が明確になるボリューム（1セクションあたり300〜500文字程度）を維持します。

### ✅ 記事の「体温」
- 冒頭と結びには、必ず「私（さしし）」の最近の釣行の感想や、読者への熱いメッセージを一言添えて、血の通った文章にしてください。
- 専門用語（棚、潮汐、バチ抜け等）は使いつつ、初心者が置いてけぼりにならないよう文脈で補足します。

## 🛠️ Markdown 記法ルール

### ✅ 強調 (Emphasis)
- **Production Rules**: 本番用（`src/content/`）では、強調に必ず `<strong>強調内容</strong>` タグを使用してください。`**` は使用禁止です。
- **SEO Strategy**: 1段落あたり1箇所程度の自然な密度を維持し、重要キーワードを強調します。

### ✅ 1文1段落ルール (Mobile First)
- **1 Sentence = 1 Paragraph**: 文章のリズムを作り、モバイルでの圧迫感を減らすため、**1つの文が終わるごとに必ず空行を1行挟んでください**。

### ✅ スペースの扱い
- 日本語と英数字・記号の間には **スペースを入れない**。
- 例：`AstroSphereで構築する9月のレポート。`

## 🛠️ コンポーネント使用ガイドライン

### ✅ 内部リンク (BlogCard)
- 形式： `<BlogCard slug="slug" />`
- **【重要: リンクの捏造禁止】**: 必ず `.workspace/` 配下のインデックスファイルを参照し、実在するスラッグを指定してください。
- 記事冒頭に `import BlogCard from "@components/BlogCard.astro";` が必要です。

### ✅ 道具紹介 (TackleCard)
- 形式： `<TackleCard id="category/id" />`
- **【重要: IDの捏造禁止】**: `src/content/affiliates/` 配下にある実際のディレクトリ名とID（ファイル名）を組み合わせて指定してください。
- 記事冒頭に `import TackleCard from "@components/TackleCard.astro";` が必要です。

## 🖼️ 画像作成・配置ワークフロー

1. **[Plan]**: 記事内にプロンプトをコメントアウト `{/* Image Prompt (...) : ... */}` で記述。
2. **[Execute]**: `generate_image` ツールを使用して `cover.png`等を作成。
