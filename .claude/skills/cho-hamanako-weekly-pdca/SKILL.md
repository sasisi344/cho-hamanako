# Cho! Hamanako — 週間 PDCA ワークフロー

## 概要

週次のアクセスデータ（GA4 / GSC）を読み込み、Check（分析・要因仮説）と Act（優先度付きタスク）を生成して `.workspace/.task/W{nn}-task.md` に書き出す。

---

## Step 1: データ取得

### ファイルパス規則

```
.workspace/access-data/2026/w{nn}/
  chohama-gsc-w{nn}.csv   ← GSC（クリック・表示回数・CTR・順位）
  chohama-ga4-w{nn}.csv   ← GA4（ユーザー・PV・直帰率・滞在時間）
```

ユーザーが週番号を指定したらそのフォルダを読む。指定がなければ `.workspace/access-data/2026/` 内で最新フォルダ（数値が最大の `w{nn}`）を選ぶ。

### GSC CSV 構造

| 列 | 内容 |
|----|------|
| ランディング ページ + クエリ文字列 | URL（行 = ランディングページ） |
| クリック数 | オーガニッククリック |
| 表示回数 | インプレッション |
| CTR | クリック率（小数） |
| 平均掲載順位 | 検索順位（小さいほど上位） |
| アクティブ ユーザー | GA4 側のアクティブユーザー数 |

最初の数行にコメントヘッダー（`# ---`）がある。スキップして8行目以降を読む。
2行目（ヘッダー行）の後の最初のデータ行（`,119,...,総計`）が全体サマリー。

### GA4 CSV 構造

複数セグメント列が横並び（すべてのユーザー / オーガニック / モバイル / ウェブ）。
- 総ユーザー数・アクティブユーザー・表示回数・平均エンゲージメント時間・直帰率
- 行 = ページタイトル × URL × 新規/既存

最初の7行はコメントヘッダー。9行目がサマリー行（`,,, 218, 206, ...`）。

---

## Step 2: 指標の読み取り（サマリー行）

### GA4 サマリーから抽出

- **総ユーザー数**（すべてのユーザー）
- **表示回数**（= PV）
- **直帰率**（小数 → %変換）
- **セッションあたりの平均エンゲージメント時間**（秒）

### GSC サマリーから抽出

- **総クリック数**
- **総表示回数**
- **全体 CTR**
- **全体平均掲載順位**

---

## Step 3: GSC 4-Box 分析

表示回数 ≥ 5 のページを対象に分類する。

| 区分 | 条件 | アクション |
|------|------|----------|
| **優先A** 高順位×低CTR | 順位 ≤ 5 かつ CTR < 全体CTR | タイトル・meta desc 改善 |
| **優先B** 低順位×高CTR | 順位 > 8 かつ CTR ≥ 全体CTR | 本文強化・内部リンク追加 |
| **維持** 高順位×高CTR | 順位 ≤ 5 かつ CTR ≥ 全体CTR | 現状維持・内部リンク受け側に活用 |
| **要観察** 低順位×低CTR | 順位 > 8 かつ CTR < 全体CTR | コンテンツ見直しまたは放置 |

「未クリック高順位（CTR 0%）」があれば別途抽出して Act に追加。

---

## Step 4: GA4 LP Top 分析

GA4 CSV から「すべてのユーザー」セグメントの表示回数上位5ページを抽出。
各ページについて記録する：

- URL
- PV（表示回数）
- 直帰率
- 平均エンゲージメント時間

直帰率が 90% 超えのページは「リライト候補」として Act に追記。

---

## Step 5: Check セクションの生成

```markdown
### Check（W{nn} 分析・要因仮説）

- **数値変化**: ユーザー数 {前週} → {今週}（{±%}）、PV {前週} → {今週}（{±%}）。
  CTR {前週%} → {今週%}（{±%}）、平均滞在時間 {前週} → {今週}（{±%}）、
  直帰率 {前週%} → {今週%}（{±%}）、掲載順位 {前週} → {今週}（{±}）。
- **LP Top流入**: （上位ページとPV）
- **要因仮説**: （データから読み取れる仮説）
- ⚠️ **データ制約**: （比較元・計算上の注意点があれば記載）
```

前週データがない場合は「前週比 データなし（初回計測）」と記載。

---

## Step 6: Act セクションの生成

```markdown
### Act（W{nn} → W{nn+1} タスク）

- [ ] {優先A ページ URL}（表示回数{n}・順位{x}・CTR{y}%）→ タイトル・meta desc 改善（優先度：高）
- [ ] {優先B ページ URL}（表示回数{n}・順位{x}・CTR{y}%）→ 本文強化・内部リンク追加（優先度：中）
- [ ] {直帰率90%超ページ} → 導入文・タイトルのリライト（優先度：中）
- [ ] 前週 Act の未完了タスクを task.md から引き継ぎ（必要なら記載）
```

タスクは **優先度：高 → 中 → 低** の順に並べる。

---

## Step 7: ファイル出力

出力先: `.workspace/.task/W{nn}-task.md`

```markdown
# W{nn} アクセス解析から次にやるタスク

主にサイト改善の項目が該当する。

### Check（W{nn} 分析・要因仮説）
{Step 5 の内容}

### Act（W{nn} → W{nn+1} タスク）
{Step 6 の内容}
```

ファイルが既に存在する場合は上書きせず、ユーザーに確認する。

---

## データ制約の扱い（既知の問題）

| 制約 | 対処 |
|------|------|
| GA4 に「デフォルトチャネル」次元がない | チャネル別セグメントは空欄と明記 |
| 指標粒度が週をまたいで異なる（セッション vs ページ） | 前週比滞在時間の比較には注意書きを追加 |
| 新規 vs リピーターの UU は重複計上になる | PV 加重平均の直帰率を代替値として使用 |
| 表示回数が少ない（< 5）URL | 4-Box 分析から除外し、「サンプル不足」と記載 |
