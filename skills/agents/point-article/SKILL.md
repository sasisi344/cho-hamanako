# 📍 Personality: Point Article (釣り場紹介)

## 🎭 性格：正確さと利便性の提供
釣り場の基本データ（座標・設備）を正確に構造化し、ユーザーが「今すぐ現地に行ける」確信を持てる情報を提供します。

## 📍 表記統一ルール (優先表記)
地名の表記揺れを厳格に防ぎ、サイト内検索とGEO（位置情報検索）の精度を高めます。

| 読み | 優先表記 | 避けるべき不一致表記 |
| :--- | :--- | :--- |
| すなあげば | **砂揚げ場** | 砂揚場 |
| あみほしば | **網干場** | 網干 |
| いまぎれぐち | **今切口** | 今切 |
| あらいべんてんうみづりこうえん | **新居弁天海釣公園** | 海釣り公園、新居海釣公園 |
| まいさかづつみ | **舞阪堤** | 舞阪堤防 |
| こうろ | **ミオ筋（航路）** | みお筋、澪筋 |

## 🛠️ フロントマター定義（必須）
カテゴリー `points` の性格として、以下の詳細なフィールド定義を厳守してください。

### A. 共通フィールド
```yaml
title: "記事タイトル" # 32文字程度、地名＋ベネフィット（例：～表浜名湖の極上ポイント～）
summary: "要約" # 70-100文字
category: "points"
tags: ["エリア名", "特徴タグ"] # エリア（表・中・奥）＋施設など
```

### B. ポイント固有フィールド（構造化データ）
```yaml
location:
  name: "ポイント正式名称"
  address: "住所（静岡県浜松市...）"
  lat: 34.680353 # 数値必須
  lng: 137.603328 # 数値必須
  googleMapUrl: "GoogleマップURL"

fishinginfo:
  difficulty: "Beginner" | "Intermediate" | "Advanced"
  familyFriendly: boolean # ファミリー向けフラグ
  bestSeason: ["春", "夏", "秋", "冬"]
  methods: ["サビキ釣り", "投げ釣り"]
  targetFish: ["アジ", "サバ"]

facilities:
  parking: true
  parkingFee: "無料" または "金額目安"
  toilet: true
  convenienceStore: "あり" または "店名"
  nightFishing: true
```

## 📝 記事構成プロセス

### ステップ1：プランニング (SXO重視)
- 検索意図（例：「浜名湖 釣り場 駐車場あり」）を想定し、冒頭で利便性を強調する。
- 設備情報は必ず `data-set/infrastructure-master.json` のマスターデータを正とする。

### ステップ2：標準構成（本文）
1.  **導入 (Intro)**: `『釣！浜名湖』をご覧いただきありがとうございます！` から開始し、その場所が「誰にとって最適か」を即座に伝える。
2.  **基本データ (H2)**: ポイントの基本情報。Google Mapの `<iframe>` 埋め込みと、設備情報の詳細（駐車場、トイレ、近隣店舗）。
3.  **地形と特徴 (H3)**: 水深、潮通し、底の質（砂、根、カキ棚）。
4.  **シーズン・対象魚 (H2)**: 季節ごとの狙い目。
5.  **釣り方別攻略 (H2)**: 
    - **エサ釣り**: 推奨タックル (`<TackleCard>`) とエサ。
    - **ルアー釣り**: ヒットパターン、推奨ルアー。
6.  **周辺観光 (H2)**: 家族連れ向けの代替案（グルメ、温泉）。
7.  **まとめ (H2)**: マナー協力の呼びかけ（`<div class="note warn">` 使用）。

---

## ⚠️ 重要ルール
- **HTML Callouts**: 強調は `<div class="note">`、警告は `<div class="note warn">` を使用。
- **座標精度**: GPS座標は Google Map から正確に取得し、数値型で格納する。
- **SSOT**: 設備情報の数値や詳細は必ず `data-set/infrastructure-master.json` を参照し、推測で書かないこと。
